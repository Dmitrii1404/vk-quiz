# VK Quiz - Пояснительная записка

Интерактивная платформа для проведения квизов в реальном времени.  
Репозиторий: https://github.com/Dmitrii1404/vk-quiz  
Макеты: https://www.figma.com/design/FajhMaChum348MAJaQRokW

---

## 1. Основная задача

Разработать MVP веб-приложения для проведения викторин в реальном времени, которое позволяет:

- **Организаторам** - создавать квизы, настраивать вопросы и запускать сессии, к которым участники подключаются по короткому коду комнаты.
- **Участникам** - входить в активный квиз без регистрации, отвечать на вопросы пока они отображаются на экране, видеть свои результаты и место в рейтинге.

Ключевое требование - **взаимодействие в реальном времени**: каждый вопрос появляется у всех участников одновременно, ответы принимаются только пока вопрос активен, а лидерборд обновляется сразу после завершения каждого вопроса.

**Обоснование выбора стека:**

| Задача | Решение | Обоснование |
|--------|---------|-------------|
| UI-фреймворк | React 18 | Компонентная модель идеальна для динамичных состояний игры (лобби -> вопрос -> результат). Большая экосистема, поддержка хуков для управления WebSocket-состоянием. |
| Сервер | Node.js + Express | Единый язык (JS) на фронте и бэке ускоряет разработку. Express минималистичен и хорошо интегрируется с Socket.IO. |
| Реальное время | Socket.IO | Надёжная абстракция над WebSocket с автоматическим переподключением, комнатами (rooms) и fallback на long-polling. Идеально для игровых сессий. |
| База данных | SQLite (better-sqlite3) | Для MVP не нужен отдельный сервер БД. SQLite - встраиваемая БД с полным SQL, синхронным API и нулевой настройкой. Данные сохраняются между перезапусками. |
| Аутентификация | JWT + bcrypt | Stateless-токены не требуют хранения сессий на сервере. bcrypt обеспечивает безопасное хэширование паролей. |

## 2. Реализованный проект - сценарии использования

### Сценарий A: Организатор создаёт и проводит квиз

1. Организатор регистрируется / входит в аккаунт.
2. На дашборде нажимает **«+ Создать квиз»** и заполняет форму:
    - Название, категория (из 8 вариантов), время на вопрос (5–120 сек).
    - Добавляет вопросы: текст, опциональное изображение по URL, тип (**одиночный** или **множественный** выбор), от 2 до 6 вариантов ответа. Правильные варианты отмечаются интерактивно.
3. Сохраняет квиз. На дашборде он появляется в списке с кнопками **«Запустить»** и **«Изменить»**.
4. Нажимает **«Запустить»** - сервер создаёт комнату с уникальным **6-значным кодом** (буквы + цифры, без похожих символов).
5. Организатор попадает в **лобби**: видит большой код комнаты, список подключившихся участников в реальном времени.
6. Когда все зашли - нажимает **«Начать квиз»**. Квиз стартует у всех участников одновременно.
7. Для каждого вопроса:
    - Видит вопрос, таймер обратного отсчёта и **живые бары** - сколько человек выбрали каждый вариант.
    - Может дождаться таймера или принудительно завершить вопрос кнопкой.
8. После каждого вопроса - автоматический показ правильных ответов и промежуточного лидерборда.
9. По завершении всех вопросов - финальный лидерборд. История сессии сохраняется в дашборде.

### Сценарий B: Участник проходит квиз

1. Открывает `/join` (ссылка доступна в навбаре без авторизации).
2. Вводит **6-значный код** комнаты - сразу видит название квиза и количество участников.
3. Вводит имя (если не авторизован) или входит с аккаунтом (результаты сохранятся в истории).
4. Попадает в **лобби ожидания** - видит список всех участников, свой ник выделен.
5. Когда организатор стартует - получает вопросы:
    - Одиночный выбор: клик по варианту -> автоматическая отправка через 400 мс.
    - Множественный выбор: отмечает все нужные варианты -> кнопка **«Подтвердить»**.
    - После ответа видит результат (+очки за правильный ответ, бонус за скорость).
    - Если не успел ответить - вопрос закрывается по таймеру.
6. После каждого вопроса - показ правильных ответов и лидерборда со своим местом.
7. По итогу квиза - финальный экран с местом и количеством очков.

### Сценарий C: Переподключение

Если участник или организатор потерял соединение в середине квиза - при повторном входе на страницу они автоматически переподключаются к активному вопросу. Состояние игры хранится на сервере в памяти, данные участника - в БД.

### Система очков

- За правильный ответ: **500 базовых очков + до 500 бонусных** за скорость (чем быстрее - тем больше).
- За неправильный ответ или отсутствие ответа: **0 очков**.
- Формула бонуса: `round((timeLeft / totalTime) * 500)`.

## 3. Бэкенд

### Стек

| Пакет | Версия | Роль |
|-------|--------|------|
| Node.js | ≥18 | Среда выполнения |
| Express | ^4.18 | HTTP-сервер, REST API |
| Socket.IO | ^4.7 | WebSocket-сервер, игровые события |
| better-sqlite3 | ^9.4 | SQLite, синхронный ORM |
| jsonwebtoken | ^9.0 | Выпуск и верификация JWT |
| bcryptjs | ^2.4 | Хэширование паролей |
| uuid | ^9.0 | Генерация ID записей |
| multer | ^1.4 | Загрузка файлов (зарезервировано) |

### Схема базы данных

```
users            - аккаунты пользователей (id, username, email, password_hash)
quizzes          - квизы (id, title, category, time_per_question, organizer_id)
questions        - вопросы квиза (id, quiz_id, text, image_url, type, order_index)
options          - варианты ответов (id, question_id, text, is_correct)
rooms            - игровые комнаты (id, quiz_id, code, status, current_question_index)
room_participants - участники комнаты (id, room_id, user_id, username, score)
answers          - ответы участников (id, room_id, question_id, selected_option_ids, is_correct, points)
```

### REST API

```
POST /api/auth/register        - регистрация
POST /api/auth/login           - вход, возвращает JWT
GET  /api/auth/me              - профиль по токену

GET  /api/quizzes/my           - мои квизы (с числом вопросов)
GET  /api/quizzes/history      - история участия в квизах
GET  /api/quizzes/:id          - квиз с вопросами и вариантами
POST /api/quizzes              - создать квиз
PUT  /api/quizzes/:id          - обновить квиз
DEL  /api/quizzes/:id          - удалить квиз

POST /api/rooms                - создать комнату для квиза
GET  /api/rooms/:code          - информация о комнате по коду
GET  /api/rooms/:code/results  - финальные результаты
GET  /api/rooms/my/hosted      - история проведённых комнат
```

### WebSocket-события

**Клиент -> Сервер:**

| Событие | Данные | Описание |
|---------|--------|----------|
| `room:join` | code, username, token? | Участник входит в комнату |
| `room:host` | code, token | Организатор подключается к своей комнате |
| `quiz:start` | code, token | Запуск квиза |
| `quiz:next` | code, token | Принудительно завершить текущий вопрос |
| `quiz:answer` | code, participantId, optionIds[] | Отправить ответ |

**Сервер -> Клиент:**

| Событие | Данные | Описание |
|---------|--------|----------|
| `room:joined` | room, participantId, participants[] | Успешное подключение участника |
| `room:host_joined` | room, quiz, participants[] | Успешное подключение организатора |
| `room:participant_joined` | participant, participants[] | Новый участник зашёл в лобби |
| `quiz:started` | total | Квиз начался |
| `quiz:question` | question, index, total, timeLeft, timeLimit | Новый вопрос (без правильных ответов) |
| `quiz:answer_result` | isCorrect, points, correctIds | Результат ответа (только отвечавшему) |
| `quiz:answer_count` | counts, answeredCount | Статистика ответов (всем) |
| `quiz:question_end` | correctIds, answerCounts | Вопрос завершён, раскрытие ответов |
| `quiz:leaderboard` | leaderboard[], isLast | Промежуточный рейтинг |
| `quiz:finished` | leaderboard[] | Квиз завершён, финальный рейтинг |

### Игровая логика (server.js)

Состояние каждой активной игры хранится в `Map<roomCode, gameState>` в памяти процесса:
- `questions[]` - все вопросы с вариантами
- `currentIndex` - текущий вопрос
- `answeredParticipants` - Set ID тех, кто уже ответил
- `answerCounts` - счётчик выборов каждого варианта
- `questionEndTime` - момент истечения таймера
- `timer` - ссылка на `setTimeout` для автоперехода

После ответа всех участников вопрос завершается досрочно. По истечении таймера - автоматически. Организатор может завершить вопрос вручную.

## 4. Фронтенд

### Стек

| Пакет | Версия | Роль |
|-------|--------|------|
| React | ^18.2 | UI-фреймворк |
| React Router | ^6.22 | Клиентский роутинг |
| Socket.IO Client | ^4.7 | WebSocket-клиент |
| Axios | ^1.6 | HTTP-запросы к API |
| Vite | ^5.1 | Сборщик, dev-сервер с HMR |

Сторонние UI-библиотеки не использовались. Все компоненты и стили написаны с нуля.

### Структура

```
src/
├── contexts/
│   └── AuthContext.jsx     - глобальное состояние авторизации (JWT в localStorage)
├── components/
│   ├── Navbar.jsx          - навигационная панель
│   ├── Timer.jsx           - SVG-таймер с анимацией и сменой цвета (зелёный -> жёлтый -> красный)
│   └── Leaderboard.jsx     - список участников с медалями и подсветкой текущего игрока
├── pages/
│   ├── Login.jsx           - форма входа
│   ├── Register.jsx        - форма регистрации
│   ├── Dashboard.jsx       - личный кабинет: квизы, история проведённых, история участия
│   ├── CreateQuiz.jsx      - конструктор квиза с динамическими вопросами
│   ├── EditQuiz.jsx        - обёртка над CreateQuiz с предзагрузкой данных
│   ├── HostRoom.jsx        - игровой экран организатора (лобби -> вопрос -> результаты)
│   ├── JoinRoom.jsx        - вход по коду с превью комнаты
│   └── PlayRoom.jsx        - игровой экран участника (лобби -> вопрос -> результаты)
├── App.jsx                 - роутер, защищённые маршруты
├── socket.js               - singleton-инстанс Socket.IO клиента
└── index.css               - дизайн-система: CSS-переменные, утилитарные классы
```

### Что умеет фронтенд

- **Аутентификация:** JWT хранится в `localStorage`, передаётся в каждый запрос через `axios.defaults.headers`. При открытии приложения токен автоматически проверяется через `/api/auth/me`.
- **Конструктор квизов:** добавление/удаление/перемещение вопросов, переключение типа (одиночный/множественный), drag-like управление порядком кнопками ↑↓, интерактивные чекбоксы правильных ответов, предпросмотр изображения по URL.
- **Игровой клиент участника:** автоотправка при одиночном выборе (400 мс дебаунс), ручное подтверждение для множественного, визуальная обратная связь (правильно/неправильно + очки), повторное подключение при разрыве соединения.
- **Игровой клиент организатора:** живые бары ответов с процентами, счётчик ответивших, ручное завершение вопроса.
- **Роутинг:** защищённые маршруты (`RequireAuth`), редирект авторизованных с `/login` на `/dashboard`, маршруты для гостей (join, play).
- **Дизайн:** адаптивная вёрстка (мобильная сетка ответов 1 колонка, десктоп 2 колонки), цветовая схема VK (`#0077FF`), SVG-таймер с анимированным кольцом.

## 5. Запуск проекта

### Требования
- **Node.js** v18+ ([nodejs.org](https://nodejs.org/))

### Шаг 1 - Установка зависимостей

```bash
# Бэкенд
cd backend
npm install

# Фронтенд
cd ../frontend
npm install
```

> Если `better-sqlite3` не устанавливается на Windows - нужны [Build Tools для Visual Studio](https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio-2022) (компонент «Desktop development with C++»).

### Шаг 2 - Запуск бэкенда

Открыть первый терминал:

```bash
cd backend
npm run dev
```

Сервер запустится на **http://localhost:3001**  
База данных создаётся автоматически в `backend/data/quiz.db`.

### Шаг 3 - Запуск фронтенда

Открыть второй терминал:

```bash
cd frontend
npm run dev
```

Приложение откроется на **http://localhost:5173**

### Быстрый тест

1. Зарегистрируйтесь на `http://localhost:5173/register`
2. Создайте квиз с несколькими вопросами
3. Нажмите «Запустить» - получите код комнаты
4. В другой вкладке (или браузере) откройте `http://localhost:5173/join`
5. Введите код и имя участника -> дождитесь старта -> отвечайте на вопросы
